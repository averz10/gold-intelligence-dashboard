# Deployment Guide - V10 Supabase Logging

## Recommended deployment route

Use:

```text
GitHub + Streamlit Community Cloud + Supabase
```

This gives you a permanent web link and persistent cloud history.

---

## Part 1 - Supabase setup

### Step 1 - Create a Supabase project

1. Go to Supabase.
2. Create a free project.
3. Wait for it to finish provisioning.

### Step 2 - Create the score_history table

In Supabase:

```text
SQL Editor
→ New query
```

Paste this SQL and run it:

```sql
create table if not exists public.score_history (
    id bigint generated by default as identity primary key,
    created_at timestamptz default now(),

    timestamp_utc timestamptz,
    supporting_gold_pct numeric,
    opposing_gold_pct numeric,
    bias text,
    confidence text,
    edge_pts numeric,

    spot_xauusd_latest numeric,
    spot_xauusd_change_pct numeric,
    gold_futures_latest numeric,
    gold_futures_change_pct numeric,

    dxy_latest numeric,
    dxy_change_pct numeric,
    us10y_latest numeric,
    us10y_change_bps numeric,
    real_yield_latest numeric,
    real_yield_change_bps numeric,
    spx_change_pct numeric,
    vix_change_pct numeric,

    dxy_state text,
    us10y_state text,
    real_yields_state text,
    fed_tone_state text,
    fed_tone_label text,
    fed_tone_confidence numeric,
    geopolitics_state text,
    risk_sentiment_state text,
    gold_news_state text,
    gold_news_articles numeric,

    notes text,
    price_1h_later numeric,
    price_4h_later numeric,
    bias_result_1h text,
    bias_result_4h text
);
```

### Step 3 - Get Supabase credentials

In Supabase:

```text
Project Settings
→ API
```

Copy:

```text
Project URL
anon public key
```

For a private Streamlit app, the service role key can also work, but start with the anon key only if your policies allow insert/select.

### Step 4 - Enable access

Simplest testing route:

1. In Supabase, go to **Table Editor**
2. Open `score_history`
3. Use **RLS disabled** for early testing only

More secure route:

- Enable Row Level Security
- Add insert/select policies for the anon role

For this personal MVP, you can test with RLS disabled first, then tighten later.

## Part 2 - Local Streamlit secrets

Create this file locally:

```text
.streamlit/secrets.toml
```

Add:

```toml
supabase_url = "https://YOUR_PROJECT_ID.supabase.co"
supabase_key = "YOUR_SUPABASE_ANON_OR_SERVICE_KEY"
supabase_table_name = "score_history"
```

Do not upload this file to GitHub.

The `.gitignore` already excludes `.streamlit/secrets.toml`.

---

## Part 3 - Test locally

Run:

```cmd
python -m streamlit run app.py
```

Then:

1. Press **Fetch market data now**
2. Press **Fetch Fed tone now**
3. Press **Fetch gold news now**
4. Turn on **Also save to Supabase**
5. Press **Save current bias snapshot**
6. Check Supabase Table Editor for the new row

---

## Part 4 - Deploy to Streamlit Cloud

Upload these files to GitHub:

```text
app.py
requirements.txt
README.md
DEPLOYMENT_GUIDE.md
RUN_DASHBOARD_WINDOWS.bat
.gitignore
.streamlit/config.toml
```

Do not upload:

```text
.streamlit/secrets.toml
data/score_history.csv
old zip files
```

Then in Streamlit Community Cloud:

```text
App settings
→ Secrets
```

Paste:

```toml
supabase_url = "https://YOUR_PROJECT_ID.supabase.co"
supabase_key = "YOUR_SUPABASE_ANON_OR_SERVICE_KEY"
supabase_table_name = "score_history"
```

Reboot the app.
